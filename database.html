Key Database Concepts

Data vs Information

Data: Raw facts and figures without context (e.g., "25", "Ahmed")

Information: Data that has been processed and given meaning (e.g., "Ahmed scored 25 in the test")

Hint بالعربية: البيانات هي المادة الخام بدون معنى، بينما المعلومات هي بيانات تمت معالجتها وأصبحت مفيدة

Database (DB)

An organized collection of related data stored efficiently in one place

Hint بالعربية: قاعدة البيانات هي مجموعة منظمة من البيانات المتعلقة ببعضها والمخزنة في مكان واحد

DBMS

Database Management System - Software to create, manage, and manipulate databases (e.g., MySQL, Oracle)

Hint بالعربية: نظام إدارة قواعد البيانات - برنامج يستخدم لإنشاء وإدارة قواعد البيانات

Data Abstraction

Hiding complex storage details from users, allowing them to interact with data easily

Hint بالعربية: تجريد البيانات - إخفاء تفاصيل التخزين المعقدة عن المستخدمين

Three-Schema Architecture

Internal (physical storage), Logical (database structure), External (user views)

Hint بالعربية: هيكل المخططات الثلاث - الداخلي (التخزين)، المنطقي (هيكل البيانات)، الخارجي (وجهات نظر المستخدم)

Entity

A real-world object or concept we want to store data about (e.g., Student, Course)

Hint بالعربية: الكيان - شيء في العالم الحقيقي نريد تخزين بيانات عنه

Attribute

A property or characteristic of an entity (e.g., Student ID, Course Name)

Hint بالعربية: السمة - خاصية أو صفة للكيان

Primary Key (PK)

A unique identifier for each record in a table (cannot be NULL or duplicate)

Hint بالعربية: المفتاح الأساسي - معرف فريد لكل سجل في الجدول

Foreign Key (FK)

A field in one table that refers to the Primary Key in another table, creating a relationship

Hint بالعربية: المفتاح الأجنبي - حقل في جدول يشير إلى المفتاح الأساسي في جدول آخر

Cardinality

Describes the numerical relationship between entities (1:1, 1:M, M:M)

Hint بالعربية: الكاردينالية - تصف العلاقة العددية بين الكيانات
